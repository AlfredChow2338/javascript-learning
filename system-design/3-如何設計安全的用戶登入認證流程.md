# 如何設計安全的用戶登入認證流程

設計用戶登入後核實身份，認證權限確定可載入資源，利用不同的authentication方法安全的前後端交互。

## 概述

本文件詳細說明使用 JWT (JSON Web Token) 配合 Refresh Token 和 Access Token 的完整認證流程，包括前後端交互的每個步驟。

## 核心概念

### Token 類型

1. **Access Token**
   - 短期有效（通常 15 分鐘 - 1 小時）
   - 用於訪問受保護的資源
   - 存儲在內存中（推薦）或 httpOnly cookie
   - 過期後需要通過 Refresh Token 獲取新的 Access Token

2. **Refresh Token**
   - 長期有效（通常 7-30 天）
   - 用於刷新 Access Token
   - 必須存儲在 httpOnly cookie 或安全的存儲中
   - 只能通過專門的 refresh endpoint 使用

## 完整流程

### 1. 用戶登入 (Login)

#### 前端請求
```javascript
// 前端發送登入請求
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "userPassword123"
}
```

#### 後端處理
```javascript
// 後端驗證用戶憑證
app.post('/api/auth/login', async (req, res) => {
  const { email, password } = req.body;
  
  // 1. 驗證用戶是否存在
  const user = await User.findOne({ email });
  if (!user) {
    return res.status(401).json({ error: 'Invalid credentials' });
  }
  
  // 2. 驗證密碼
  const isValidPassword = await bcrypt.compare(password, user.password);
  if (!isValidPassword) {
    return res.status(401).json({ error: 'Invalid credentials' });
  }
  
  // 3. 生成 Access Token (短期)
  const accessToken = jwt.sign(
    { 
      userId: user.id, 
      email: user.email,
      role: user.role 
    },
    process.env.ACCESS_TOKEN_SECRET,
    { expiresIn: '15m' } // 15分鐘
  );
  
  // 4. 生成 Refresh Token (長期)
  const refreshToken = jwt.sign(
    { userId: user.id },
    process.env.REFRESH_TOKEN_SECRET,
    { expiresIn: '7d' } // 7天
  );
  
  // 5. 將 Refresh Token 存儲到資料庫（用於撤銷和驗證）
  await RefreshToken.create({
    userId: user.id,
    token: refreshToken,
    expiresAt: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
  });
  
  // 6. 設置 httpOnly Cookie（更安全）
  res.cookie('refreshToken', refreshToken, {
    httpOnly: true,        // 防止 XSS 攻擊
    secure: true,          // HTTPS only
    sameSite: 'strict',    // CSRF 保護
    maxAge: 7 * 24 * 60 * 60 * 1000 // 7天
  });
  
  // 7. 返回 Access Token（存儲在內存中）
  res.json({
    accessToken,
    user: {
      id: user.id,
      email: user.email,
      role: user.role
    }
  });
});
```

#### 前端處理
```javascript
// 前端接收並存儲 Token
const response = await fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email, password })
});

const data = await response.json();

// 存儲 Access Token 到內存（推薦）或 localStorage
// 注意：不推薦存儲在 localStorage，因為容易受到 XSS 攻擊
// 最佳實踐：存儲在內存變量中
let accessToken = data.accessToken;

// Refresh Token 已經自動存儲在 httpOnly cookie 中
// 前端無法直接訪問，更安全
```

### 2. 訪問受保護資源 (Access Protected Resources)

#### 前端請求
```javascript
// 前端在請求頭中攜帶 Access Token
const response = await fetch('/api/protected-resource', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json'
  }
});
```

#### 後端驗證中間件
```javascript
// JWT 驗證中間件
const authenticateToken = async (req, res, next) => {
  // 1. 從 Authorization header 提取 Token
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1]; // Bearer TOKEN
  
  if (!token) {
    return res.status(401).json({ error: 'Access token required' });
  }
  
  try {
    // 2. 驗證 Access Token
    const decoded = jwt.verify(token, process.env.ACCESS_TOKEN_SECRET);
    
    // 3. 將用戶信息附加到請求對象
    req.user = decoded;
    
    // 4. 繼續處理請求
    next();
  } catch (error) {
    if (error.name === 'TokenExpiredError') {
      return res.status(401).json({ 
        error: 'Token expired',
        code: 'TOKEN_EXPIRED' 
      });
    }
    return res.status(403).json({ error: 'Invalid token' });
  }
};

// 使用中間件保護路由
app.get('/api/protected-resource', authenticateToken, (req, res) => {
  // req.user 包含解碼後的用戶信息
  res.json({ 
    message: 'Protected data',
    user: req.user 
  });
});
```

### 3. Token 刷新流程 (Token Refresh)

當 Access Token 過期時，前端需要使用 Refresh Token 獲取新的 Access Token。

#### 前端處理 Token 過期
```javascript
// 封裝的 API 請求函數，自動處理 Token 刷新
let accessToken = null;

async function apiRequest(url, options = {}) {
  // 1. 發送請求
  let response = await fetch(url, {
    ...options,
    headers: {
      ...options.headers,
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type': 'application/json'
    }
  });
  
  // 2. 如果 Token 過期，嘗試刷新
  if (response.status === 401) {
    const errorData = await response.json();
    
    if (errorData.code === 'TOKEN_EXPIRED') {
      // 3. 調用刷新 Token 的 API
      const refreshResponse = await refreshAccessToken();
      
      if (refreshResponse.success) {
        // 4. 使用新的 Access Token 重試原請求
        accessToken = refreshResponse.accessToken;
        response = await fetch(url, {
          ...options,
          headers: {
            ...options.headers,
            'Authorization': `Bearer ${accessToken}`,
            'Content-Type': 'application/json'
          }
        });
      } else {
        // 刷新失敗，需要重新登入
        window.location.href = '/login';
        throw new Error('Session expired');
      }
    }
  }
  
  return response;
}

// 刷新 Access Token
async function refreshAccessToken() {
  try {
    // Refresh Token 自動通過 Cookie 發送
    const response = await fetch('/api/auth/refresh', {
      method: 'POST',
      credentials: 'include' // 重要：包含 cookies
    });
    
    if (response.ok) {
      const data = await response.json();
      return {
        success: true,
        accessToken: data.accessToken
      };
    } else {
      return { success: false };
    }
  } catch (error) {
    return { success: false };
  }
}
```

#### 後端刷新 Token Endpoint
```javascript
// Refresh Token 端點
app.post('/api/auth/refresh', async (req, res) => {
  // 1. 從 Cookie 中獲取 Refresh Token
  const refreshToken = req.cookies.refreshToken;
  
  if (!refreshToken) {
    return res.status(401).json({ error: 'Refresh token required' });
  }
  
  try {
    // 2. 驗證 Refresh Token
    const decoded = jwt.verify(refreshToken, process.env.REFRESH_TOKEN_SECRET);
    
    // 3. 檢查 Refresh Token 是否在資料庫中存在且有效
    const storedToken = await RefreshToken.findOne({
      userId: decoded.userId,
      token: refreshToken
    });
    
    if (!storedToken || storedToken.revoked) {
      return res.status(403).json({ error: 'Invalid refresh token' });
    }
    
    // 4. 檢查是否過期（資料庫層面）
    if (new Date() > storedToken.expiresAt) {
      await RefreshToken.deleteOne({ _id: storedToken._id });
      return res.status(403).json({ error: 'Refresh token expired' });
    }
    
    // 5. 獲取用戶信息
    const user = await User.findById(decoded.userId);
    if (!user) {
      return res.status(403).json({ error: 'User not found' });
    }
    
    // 6. 生成新的 Access Token
    const newAccessToken = jwt.sign(
      { 
        userId: user.id, 
        email: user.email,
        role: user.role 
      },
      process.env.ACCESS_TOKEN_SECRET,
      { expiresIn: '15m' }
    );
    
    // 7. 可選：旋轉 Refresh Token（更安全）
    // 撤銷舊的 Refresh Token
    await RefreshToken.deleteOne({ _id: storedToken._id });
    
    // 生成新的 Refresh Token
    const newRefreshToken = jwt.sign(
      { userId: user.id },
      process.env.REFRESH_TOKEN_SECRET,
      { expiresIn: '7d' }
    );
    
    // 存儲新的 Refresh Token
    await RefreshToken.create({
      userId: user.id,
      token: newRefreshToken,
      expiresAt: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
    });
    
    // 設置新的 Refresh Token Cookie
    res.cookie('refreshToken', newRefreshToken, {
      httpOnly: true,
      secure: true,
      sameSite: 'strict',
      maxAge: 7 * 24 * 60 * 60 * 1000
    });
    
    // 8. 返回新的 Access Token
    res.json({
      accessToken: newAccessToken
    });
    
  } catch (error) {
    if (error.name === 'TokenExpiredError') {
      return res.status(403).json({ error: 'Refresh token expired' });
    }
    return res.status(403).json({ error: 'Invalid refresh token' });
  }
});
```

### 4. 登出流程 (Logout)

#### 前端登出
```javascript
async function logout() {
  // 1. 清除本地 Access Token
  accessToken = null;
  
  // 2. 調用後端登出 API（撤銷 Refresh Token）
  await fetch('/api/auth/logout', {
    method: 'POST',
    credentials: 'include'
  });
  
  // 3. 重定向到登入頁面
  window.location.href = '/login';
}
```

#### 後端登出處理
```javascript
app.post('/api/auth/logout', authenticateToken, async (req, res) => {
  const refreshToken = req.cookies.refreshToken;
  
  if (refreshToken) {
    // 撤銷 Refresh Token（標記為已撤銷或刪除）
    await RefreshToken.updateOne(
      { token: refreshToken },
      { revoked: true }
    );
  }
  
  // 清除 Cookie
  res.clearCookie('refreshToken', {
    httpOnly: true,
    secure: true,
    sameSite: 'strict'
  });
  
  res.json({ message: 'Logged out successfully' });
});
```

## 安全性最佳實踐

### 1. Token 存儲

**Access Token:**
- ✅ 推薦：存儲在內存變量中（JavaScript 變量）
- ✅ 可選：存儲在 httpOnly Cookie 中
- ❌ 避免：localStorage、sessionStorage（容易受到 XSS 攻擊）

**Refresh Token:**
- ✅ 必須：存儲在 httpOnly Cookie 中
- ✅ 必須：設置 secure flag（HTTPS only）
- ✅ 必須：設置 sameSite: 'strict'（CSRF 保護）

### 2. Token 過期時間

- **Access Token**: 15 分鐘 - 1 小時（短期）
- **Refresh Token**: 7-30 天（長期）

### 3. Token 旋轉 (Token Rotation)

每次刷新時生成新的 Refresh Token，撤銷舊的：
- 防止 Refresh Token 被重複使用
- 如果 Refresh Token 被盜用，可以更快地檢測到

### 4. Token 撤銷機制

- 在資料庫中存儲 Refresh Token
- 支持撤銷特定用戶的所有 Token
- 登出時撤銷 Refresh Token

### 5. 防止攻擊

**XSS (Cross-Site Scripting):**
- 使用 httpOnly Cookie 存儲 Refresh Token
- 避免在 localStorage 存儲敏感信息
- 實施 Content Security Policy (CSP)

**CSRF (Cross-Site Request Forgery):**
- 設置 sameSite: 'strict' Cookie
- 使用 CSRF Token（對於非 GET 請求）

**Token 洩露:**
- 使用 HTTPS 傳輸
- 實施 Token 旋轉
- 監控異常的 Token 使用模式

### 6. 錯誤處理

- 不要洩露過多錯誤信息
- 統一錯誤響應格式
- 記錄安全相關的錯誤日誌

## 總結

這個認證流程提供了：

1. **安全性**：使用 httpOnly Cookie、Token 旋轉、HTTPS
2. **用戶體驗**：自動刷新 Token，無需頻繁登入
3. **可擴展性**：支持 Token 撤銷、多設備管理
4. **最佳實踐**：遵循 OAuth 2.0 和 JWT 標準

通過這個流程，可以實現一個安全、可靠的用戶認證系統。
