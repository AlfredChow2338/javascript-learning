['1', '2', '3'].map(parseInt) 的結果
結果：[1, NaN, NaN]

為什麼？
Array.prototype.map 傳給回呼函式的參數是：(元素, 索引, 陣列)。

你寫的是 .map(parseInt)，等同於：

['1', '2', '3'].map((element, index, array) => parseInt(element, index, array))
而 parseInt 只會用到前兩個參數：(字串, 進位 radix)。

所以實際上是：

索引 呼叫 說明
0 parseInt('1', 0) radix 0 會被當成「自動判斷」，十進位字串 '1' → 1
1 parseInt('2', 1) radix 必須在 2～36 之間，1 不合法 → NaN
2 parseInt('3', 2) 二進位只允許 0、1，'3' 無效 → NaN
若想要 [1, 2, 3]
不要直接把 parseInt 當成 map 的回呼，可以：

['1', '2', '3'].map(Number)
// 或
['1', '2', '3'].map((s) => parseInt(s, 10))
// 或
['1', '2', '3'].map((s) => parseInt(s))
重點：map 會多傳 index，而 parseInt 的第二個參數是進位，兩者湊在一起就會出現這個經典陷阱。
